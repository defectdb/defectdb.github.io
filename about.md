DefectDB is an open, structured catalogue of software defects — the recurring
ways code goes wrong, organised by language and theme. Each entry sets out the
defect's impact, its security and denial-of-service potential, and how to catch
it.

The project is built and maintained by the team behind
[interpretica.io](https://interpretica.io). We build developer tooling for code
quality and security: static code analyzers, automated testing systems, and
vulnerability detection. DefectDB grew directly out of that work — it documents,
in plain language, the classes of defects our analysis tools reason about, so
the knowledge is useful on its own.

If static analysis, test automation, or security tooling is your kind of thing,
come and say hello at [interpretica.io](https://interpretica.io).
